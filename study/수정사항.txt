[수정 16] bcrypt + passlib 5.0 호환 문제
----------------------------------------------------------------

문제:
  uvicorn 실행 후 로그인/회원가입 API 호출 시
  "AttributeError: module 'bcrypt' has no attribute '__about__'" 발생.

원인:
  passlib이 bcrypt의 버전 정보를 bcrypt.__about__.__version__으로 읽는데
  bcrypt 5.0에서 __about__ 모듈이 제거됨.

수정 내용:
  - requirements.txt에서 passlib 제거
  - services/auth.py: passlib 완전 제거 후 bcrypt 직접 사용
    _hash(pw)   → bcrypt.hashpw(pw.encode(), bcrypt.gensalt()).decode()
    _verify(pw) → bcrypt.checkpw(pw.encode(), hashed.encode())

수정 파일: requirements.txt, services/auth.py

[수정 25] 앱 외부에서 앞으로 버튼 진입 시 화면 동결 문제
----------------------------------------------------------------

문제:
  메인 화면(/)에서 뒤로 버튼을 누르면 앱 외부(빈 탭, 이전 사이트 등)로 이동.
  그 상태에서 앞으로 버튼을 누르면 Next.js 앱으로 돌아오지만 화면이 완전히 굳어버림.
  콘솔 에러: "Router action dispatched before initialization"

원인:
  브라우저가 앞으로 버튼으로 앱에 재진입할 때 bfcache(Back-Forward Cache)에서
  페이지를 그대로 복원함. 이 시점에 Next.js 라우터가 아직 초기화되지 않은 상태에서
  브라우저 history 이벤트가 발생해 충돌.
  (수정 19에서 window.location.href → router.push()로 교체했으나,
   앱 외부 → 앱 재진입 경로는 별개의 bfcache 문제였음)

수정 내용:
  1차 시도 (useEffect 방식 — 실패):
    - BfcacheGuard.tsx를 'use client' + useEffect로 pageshow 리스너 등록
    - 문제 1: React cleanup이 bfcache 진입 전 리스너를 제거할 수 있어 불안정
    - 문제 2: bfcache가 아닌 fresh 앞으로 이동(e.persisted === false)은 감지 불가

  2차 수정 (인라인 스크립트 방식 — 해결):
    - BfcacheGuard.tsx를 서버 컴포넌트 + <script dangerouslySetInnerHTML>로 교체
    - HTML 파싱 시점에 즉시 실행 → React 초기화 이전, React lifecycle 무관
    - 두 케이스 모두 처리:
        (1) bfcache 복원: pageshow 이벤트 e.persisted === true → reload()
        (2) fresh 앞으로 이동: performance.getEntriesByType('navigation')[0]
            .type === 'back_forward' → reload()
    - reload() 후 navigation.type이 'reload'로 바뀌어 무한 루프 없음
  - layout.tsx에 <BfcacheGuard /> 전역 마운트 (모든 페이지에 자동 적용)

수정 파일: components/BfcacheGuard.tsx(신규), app/layout.tsx
