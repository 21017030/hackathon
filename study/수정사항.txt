[수정 1] AI 답변에 수식·마크다운이 깨져서 표시되는 문제
----------------------------------------------------------------

문제:
  AI가 LaTeX 수식($P(A) \ge 0$ 등)이나 볼드/**목록** 등 마크다운 형식으로
  답변을 작성해도, ChatView에서 plain text로만 렌더링되어 수식 기호가 노출됨.

원인:
  ChatView.tsx의 메시지 렌더링이 단순 <p> 태그여서 마크다운/LaTeX 처리가 없었음.

수정 내용:
  - react-markdown, remark-math, rehype-katex, katex, @tailwindcss/typography 설치
  - layout.tsx에 katex CSS import 추가
  - globals.css에 @plugin "@tailwindcss/typography" 추가
  - ChatView.tsx: AI 메시지를 <ReactMarkdown>으로 렌더링

수정 파일: layout.tsx, globals.css, ChatView.tsx, package.json

[수정 4] PDF 전용 + Gemini 멀티모달 이미지 인식 추가
----------------------------------------------------------------

배경:
  PPTX, DOCX 등 다양한 형식을 지원하고 있었으나 Gemini가 이미지·표를 읽으려면

    * 실패 시 PyMuPDF(fitz) 폴백
  - process_document_rag(): PDF 추출 로직을 _extract_pdf_gemini() 우선으로 변경
  - requirements.txt: python-pptx, python-docx 제거

수정 파일: document.py, requirements.txt

[수정 6] 동시 refresh 호출 시 API 과부하 문제
----------------------------------------------------------------

문제:
  Promise.all로 병렬 refresh 중에 추가 refresh가 호출되면 Supabase API에
  동시 요청이 몰려 오류 발생.

원인:
  이전에 useAppData.ts의 refresh가 동시 실행을 막는 로직 없이 매번 실행됨.

수정 내용:
  - Promise.all → Promise.allSettled로 변경 (하나 실패해도 나머지 반영)
  - isRefreshing ref: refresh 실행 중이면 새 호출 차단
  - pendingRefresh ref: 차단된 호출을 기억해뒀다가 완료 후 1회 재실행

수정 파일: hooks/useAppData.ts

[수정 7] RAG 품질 개선 - 쿼리 재작성 도입
----------------------------------------------------------------

문제:
  "저거 코드로 알려줘" 같은 팔로업 질문에서 지시대명사("저거")가 임베딩되어
  관련 없는 청크가 검색됨. 이전 대화 맥락이 벡터 검색에 반영되지 않았음.

수정 내용:
  - _rewrite_query(content, history): gemini-3.1-flash-lite로 팔로업 질문을
    독립적인 검색 쿼리로 변환
  - history가 없으면 원본 쿼리 그대로 사용 (불필요한 API 호출 방지)
  - ask_question, ask_about_document 모두 재작성 쿼리로 임베딩
  - 쿼리 재작성용 경량 모델(REWRITE_MODEL)을 별도 상수로 분리

수정 파일: document.py, chat.py

[수정 14] ask_about_document KeyError 수정
----------------------------------------------------------------

문제:
  POST /documents/{id}/ask 호출 시 500 오류 발생.
  스택 트레이스: KeyError: 'category' in _build_context()

원인:
  ask_about_document()가 get_relevant_chunks()를 호출하고 있었음.
  이 함수는 raw 청크(content, document_id 등)만 반환하며,
  _build_context()가 필요로 하는 filename, category, page 키가 없었음.

수정 내용:
  - ask_about_document(): get_relevant_chunks() → get_relevant_chunks_with_sources()
    (documents + categories 테이블 조인하여 출처 메타데이터 보강)

수정 파일: services/chat.py
